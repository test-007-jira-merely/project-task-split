# Batch Execution Plan: Meal Planning Application

## Summary of Research

The batch tasks file contains 3 pre-decomposed tasks for building a complete meal planning application with React, TypeScript, Vite, Tailwind CSS, and Supabase:

1. **Foundation & Infrastructure Setup** - Vite project initialization, dependencies, Tailwind configuration, folder structure, meals dataset
2. **Core Services & State Management** - Supabase services, Zustand store, React Query, custom hooks, business logic
3. **UI Components & Pages** - Complete component library, all pages, routing, animations, protected routes

**Critical Discovery:**
- All 3 tasks specify the **same branch name**: `feature/task-SCRUM-429_project`
- Tasks 2-3 are marked as "sequential (has dependencies)"
- The current structure creates a **sequential pipeline**, not parallel independent units
- The repository already has TypeScript contracts/interfaces in `.contracts-beezi/` directory

**Problem with Current Structure:**
The provided tasks cannot run in parallel with isolated worktrees and separate PRs because they all target the same branch. This violates the batch orchestration requirement that each unit must be "mergeable on its own without depending on another unit's PR landing first."

## Execution Strategy

**Approach: Sequential Execution on Single Branch**

Since all 3 tasks in the batch file target the same branch (`feature/task-SCRUM-429_project`) and are marked as having dependencies, I will execute them **sequentially** as a single unit of work:

1. **Task 1: Foundation & Infrastructure Setup**
   - Branch: `feature/task-SCRUM-429_project`
   - Initialize Vite + React + TypeScript project
   - Install all dependencies (React Router, Zustand, Supabase, Tailwind, etc.)
   - Configure Tailwind CSS with theme support
   - Create folder structure
   - Generate meals dataset
   - Set up environment configuration
   - **Deliverable:** Working development server with project scaffolding

2. **Task 2: Core Services & State Management**
   - Continue on same branch
   - Implement Supabase services (auth, favorites, history, meals)
   - Create Zustand global store
   - Set up React Query
   - Build custom hooks for data management
   - Implement meal service and utilities
   - **Deliverable:** Complete business logic layer

3. **Task 3: UI Components & Pages**
   - Continue on same branch
   - Build UI primitive components (Button, Input, Card, Modal, etc.)
   - Create layout components (Header, Sidebar, Navigation)
   - Implement meal-specific components (MealCard, MealGrid, MealDetail, etc.)
   - Build ingredient components (IngredientList, IngredientSelector)
   - Create all pages (Home, Ingredients, Favorites, History, Auth, Admin)
   - Set up React Router with protected routes
   - Add Framer Motion animations
   - **Deliverable:** Complete, working meal planning application

**Final Step:** Commit all changes and create a single PR with the complete implementation

**Rationale:**
- The provided batch file explicitly designs these as sequential tasks with dependencies
- All tasks target the same branch, making parallel execution with separate PRs impossible
- This approach follows the pre-defined structure in `.batch-tasks-group.json`

## E2E Test Recipe

Since this is a new project being built from scratch, testing will be performed incrementally after each task:

### After Task 1 (Foundation)
**Test:** Development server starts successfully
```bash
npm run dev &
sleep 5
curl -f http://localhost:5173 || exit 1
pkill -f "vite"
```

### After Task 2 (Core Services)
**Test:** TypeScript compiles without errors, services are properly structured
```bash
npm run build
```

### After Task 3 (UI Components & Pages)
**Test:** Full application functionality
```bash
# Start dev server in background
npm run dev &
sleep 5

# Check home page loads
curl -f http://localhost:5173

# Check if app renders without errors (look for React root element)
curl -s http://localhost:5173 | grep -q 'root'

# Stop dev server
pkill -f "vite"

# Build for production
npm run build
```

### Final Manual Verification
After all tasks complete, manually verify core flows:
1. Navigate to home page → see meal grid with data from dataset
2. Click on a meal → see meal details page
3. Select ingredients → see filtered meals based on matches
4. Test authentication flow (sign up/login with Supabase)
5. Add meals to favorites → verify persistence
6. Check meal history tracking

**Note:** Since this is using Supabase, either a test Supabase project will be configured or mock data will be used for development testing.

## Implementation Plan

### Task 1: Foundation & Infrastructure Setup
**Objective:** Set up complete project foundation

**Steps:**
1. Initialize Vite project with React + TypeScript template
2. Install all dependencies:
   - Core: react-router-dom, zustand, @supabase/supabase-js, @tanstack/react-query
   - Forms: react-hook-form, zod, @hookform/resolvers
   - UI: framer-motion, lucide-react, clsx
   - Styling: tailwindcss, postcss, autoprefixer
3. Configure TypeScript (tsconfig.json with strict mode, path aliases)
4. Configure Tailwind CSS with custom theme (colors, fonts, animations)
5. Create folder structure:
   - `/src/components` (ui, layout, meal, ingredient, admin)
   - `/src/pages`
   - `/src/services`
   - `/src/hooks`
   - `/src/store`
   - `/src/types`
   - `/src/utils`
   - `/src/data`
6. Generate meals dataset (25+ meals with categories, ingredients, instructions)
7. Set up environment variables template (.env.example)
8. Update package.json scripts
9. Test: Verify dev server runs successfully

**Commit:** "Initialize Vite+React+TypeScript project with complete foundation"

### Task 2: Core Services & State Management
**Objective:** Implement all business logic layer

**Steps:**
1. Set up Supabase client configuration
2. Implement authentication service (signup, login, logout, session management)
3. Create Supabase services:
   - Favorites service (add, remove, fetch favorites)
   - History service (track viewed meals, fetch history)
   - Meals service (CRUD operations)
4. Build meal service with filtering and matching logic
5. Create ingredient utilities (parsing, matching, normalization)
6. Implement Zustand store (auth state, app state, UI state)
7. Set up React Query configuration
8. Build custom hooks:
   - useAuth, useMeals, useFavorites, useHistory
   - useMealFilters, useIngredientMatch
9. Test: Verify TypeScript compiles, services are properly structured

**Commit:** "Implement core services, state management, and custom hooks"

### Task 3: UI Components & Pages
**Objective:** Build complete UI layer

**Steps:**
1. Create UI primitives:
   - Button, Input, Card, Modal, Badge, Spinner
2. Build layout components:
   - Header, Sidebar, Navigation, Footer, PageLayout
3. Implement meal components:
   - MealCard, MealGrid, MealDetail, MealFilters
   - CategoryBadge, DifficultyBadge
4. Create ingredient components:
   - IngredientList, IngredientCard, IngredientSelector, IngredientSearch
5. Build admin components (if specified in original tasks)
6. Create all pages:
   - HomePage (meal grid with filters)
   - IngredientsPage (ingredient selection + matching meals)
   - FavoritesPage (saved favorites)
   - HistoryPage (recently viewed meals)
   - AuthPage (login/signup forms)
   - AdminPage (meal management)
7. Set up React Router with routes and protected routes
8. Add Framer Motion animations (page transitions, card hover, etc.)
9. Integrate all components with services and state
10. Test: Full application flow verification

**Commit:** "Build complete UI component library and pages with routing"
